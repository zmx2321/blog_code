# React 速成 - 今日学习计划（Vue 工程师版）

> 目标：用你熟悉的 Vue3 思维，一天内建立 React 组件化开发的肌肉记忆。

---

## 一、核心对照表（背下来）

| Vue3 (你熟悉的) | React (对应写法) | 说明 |
|-----------------|------------------|------|
| `<template>` 里写 HTML | 直接在函数里 return JSX | 没有单独模板，JSX 就是「模板」 |
| `ref(0)` / `reactive({})` | `useState(0)` / 多个 `useState` | 状态：Vue 改 `.value`，React 用 setXxx |
| `computed(() => a + b)` | `useMemo(() => a + b, [a, b])` | 派生值/缓存 |
| `watch(source, cb)` | `useEffect(() => { ... }, [deps])` | 副作用、监听 |
| `defineProps` / `props.xxx` | 函数参数 `(props)` / `props.xxx` | 父传子 |
| `defineEmits` / `emit('ok')` | 父传函数 `onOk`，子调 `props.onOk()` | 子传父：传回调 |
| `v-if` / `v-show` | `{ condition && <Node />}` / 用 CSS 控制 | 条件渲染 |
| `v-for="x in list"` | `list.map(x => <Node key={x.id} />)` | 列表必须写 key |
| `v-model="text"` | `value={text} onChange={e => setText(e.target.value)}` | 受控组件 |
| `<slot />` | `props.children` 或 `props.renderFooter` | 插槽 = 传 children 或 render 函数 |

## 三、每个案例的学习要点

1. **01-jsx-basics**  
   - JSX 里用 `{表达式}`，不用 `{{ }}`。  
   - class → `className`，style 用对象。  
   - 一个组件 return 一个根节点（或 Fragment `<>...</>`）。

2. **02-state-props**  
   - `const [count, setCount] = useState(0)`：读用 count，写用 setCount。  
   - 父传子：`<Child count={count} />`；子用 `props.count`。  
   - 子传父：父传 `onIncrement={() => setCount(c => c + 1)}`，子调 `props.onIncrement()`。

3. **03-computed-effect**  
   - 派生值：`const double = useMemo(() => count * 2, [count])`。  
   - 副作用：`useEffect(() => { ... return () => {} }, [deps])`，空数组只跑一次。

4. **04-conditional-list**  
   - 条件：`{show && <div>...</div>}`。  
   - 列表：`items.map(item => <li key={item.id}>...</li>)`，key 必须稳定唯一。

5. **05-events-forms**  
   - 事件：`onClick`、`onChange` 等，传函数不是字符串。  
   - 受控输入：`value={text} onChange={e => setText(e.target.value)}`。

6. **06-todo-mini**  
   - 状态提升：多个组件要同一份数据 → 状态放在共同父组件，用 props 往下传，用回调往上改。  
   - 组件拆分：TodoList、TodoItem、TodoInput、TodoFilter，对应 Vue 里你常拆的方式。

## 四、怎么练最有效

1. **先看再写**：每个案例先跑起来（`npm run dev`），看效果和代码对应关系。  
2. **关掉代码自己写一遍**：例如只留空壳，自己实现「点击+1」「受控输入」「一个小 Todo」。  
3. **刻意用 Vue 对照**：每写一句 React，心里想「在 Vue 里我会怎么写」，巩固对照表。  
4. **晚上小结**：用 10 分钟列一张「Vue → React」对照清单（可以就是本文的表格），以后忘了就扫一眼。

## 若运行报错（esbuild / EPERM）

1. **先清空依赖再装一次**（在 `demo1` 目录下）：
   ```bash
   rm -rf node_modules
   yarn install
   yarn dev
   ```
2. **若仍报 esbuild 相关错**：本项目已锁定 `esbuild@0.20.2` 并设置了 `resolutions`，重装后一般可恢复。若你用的是 npm，可删掉 `package.json` 里的 `resolutions` 再试。
3. **若报 `listen EPERM ::1:5173`**：已在本项目里把 dev 服务器改为只监听 `127.0.0.1:5173`，正常重装后应不再出现。
